Javascript

Structure and usage within the pattern library

Node Modules

The pattern library uses several js libraries and plugins for ui design and display purposes. These are imported via the node modules require method on the custom.js file in js/custom.js: 

         
          //jQuery
          $ = jQuery = require('jquery');

          //SlideOut Nav
          var Slideout = require('slideout');

          //Materialize js from the materialize library
          var materialize = require("materialize");

          //Namespace pattern for js structure
          var namespace = require('./namespace.js');

          //Require HighCharts 
          var Highcharts = require('highcharts');

          //Page animation transitions
          var pageAnimation = require('animsition');

          // Metismenu js for vertical nav extending
          var metis = require('metismenu');

          //Syntax highlighting with Prism
          var prism = require('prismjs');
Namespace Pattern and Current Methods

The custom js file uses a basic namespacing pattern. This is used to provide some form of structure if the current codebase expands. The three main methods within the current pattern library are: init, menus and charts. They are used for basic ui rendering purposes.

A function of importance to the current template structure is mobileClearfix. It sits within the 'init' method and provides a structural check and adjustment of card sizings for the application. It is extensively commented if further information is required.
