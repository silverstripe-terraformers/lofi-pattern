CSS

Structure and Guidelines

Cards & Use Of Classes

The entire css library endeavours to be class selector based, in hyphenated format: 

.a-class {height: 0em}

In some instances chaining, multiclassing and #id's are used as css selectors. However, this non-class selector format is discouraged and only used to override or address pre-established css libraries and open source plugins. HTML elements or single hyphenated classes are the preferred css selector for the application.

CSS comments are encouraged wherever possible to explain non-evident css rules. Below we see custom selectors (.condensed & .normal) chained to the Materialzed.css library '.card' class. The custom classes '.condensed' & '.normal' are very important css classes within the application. They represent the 2 types of data card vertical size limit. This will control the size of standard cards to improve the applications design and structure within the browser. 


// Padding fix for cards with content 
// Position relative used on outer parent card div to allow for sticky bottom for card-actions

.card.condensed, 
.card.normal {
  padding: 0em;
  position: relative;
}

.card.normal {
  min-height: 44em;
  max-height: 44em;
  clear: left;
}

.card.condensed.no-visual, 
.card.condensed {
  min-height: 34em;
  max-height: 34em;
  clear: left;
}
